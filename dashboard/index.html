4-Stage IPO Recommendation Pipeline

Every pipeline stage is a decoupled, unit-testable module adhering to strict ARCHITECTURE.md contracts.

01
data_loader.py

Ingestion

Loads, parses, and validates the CSV dataset. Normalizes tokens to lowercase and replaces spaces with underscores.

02
vectorizer.py

Vectorize

Builds a corpus-fixed vocabulary. Computes TF for items/users and IDF weights. Builds the TF-IDF representation.

03
similarity_engine.py

Score

Computes Cosine Similarity between user TF-IDF vector and item matrix. Detects Cold Start condition.

04
ranker.py

Rank

Sorts matches by similarity descending, with alphabetical secondary tie-breaking. Routes fallback on Cold Start.

Cosine Similarity

cos(θ) = (A · B) / (‖A‖ × ‖B‖)

Measures the angle between the user skill vector and job role vectors. Since it normalizes by vector norms, it evaluates the proportion of skills rather than absolute counts, making it length-invariant.

User Vector A Role Vector B cos θ ≈ 0.94

TF-IDF Weighting

TF(t, d) = count(t in d) / total_terms(d)
IDF(t) = log( N / df(t) )

Term Frequency (TF) rewards repeated skill tags (stronger signal).
Inverse Document Frequency (IDF) penalizes skills common across many roles (e.g. python, sql) and amplifies specialized skills (e.g. penetration_testing, kubernetes).

python (shared by 2/3 items) log(3/2) = 0.405
kubernetes (shared by 1/3 items) log(3/1) = 1.099
