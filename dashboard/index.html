Math

Algorithm Deep-Dive

Built from first principles — every formula coded by hand, no black-box libraries.

01 Term Frequency
TF(t, d) = count(t in d) / total_terms(d)

How often a skill appears in a document — normalized by total terms. Duplicates intentionally increase TF (stronger signal).

python in [python, sql, ml] 1/3 = 0.333
python in [python, python, python] 3/3 = 1.000
02 Inverse Document Frequency
IDF(t) = log( N / df(t) )

Penalizes common terms (appear in many roles) and rewards rare, specific skills. Generic terms matter less.

python (2 of 3 items) log(3/2) = 0.405
kubernetes (1 of 3 items) log(3/1) = 1.099
03 Cosine Similarity
cos(θ) = (A · B) / (‖A‖ × ‖B‖)

Measures the angle between two skill vectors — magnitude-invariant. A long resume and a short one with identical proportions score the same. Returns 0.0 when either vector is zero (Cold Start guard).

User vector (A) Item vector (B) cos θ ≈ 0.94
04 Cold Start Handling
🧭
Skills in vocab → Cosine scores > 0 → Ranked Results
— or —
❄️
All OOV skills → Zero vector → ⚠ Trending Fallback

Never silently shows "Score: 0.00" as a recommendation — the system detects and labels the fallback explicitly.
